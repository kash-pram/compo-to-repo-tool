# Contributing to compo-to-repo-tool

Thank you for considering contributing!

## Before Contributing

1. Read the [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)
2. Review the [LICENSE](./LICENSE) - all contributions will be under MIT License
3. Read the [DISCLAIMER](./DISCLAIMER.md) to understand project risks

## How to Contribute

### Reporting Bugs

Open an issue with:
- Clear description
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Node version, Angular version)
- Screenshots if applicable

### Suggesting Enhancements

Open an issue with:
- Clear description of enhancement
- Use case / motivation
- Proposed implementation (optional)

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Update documentation
6. Commit with clear messages
7. Push to your fork
8. Open a Pull Request

### Code Standards

- Follow existing code style
- Add comments for complex logic
- Update README if adding features
- Test your changes

### Testing Contributions

**CRITICAL**: Test all changes with:
- Test GitHub account
- Dummy Angular projects
- Multiple component scenarios
- Both public and private repo creation

Never test with production code or accounts.
